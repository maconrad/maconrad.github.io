Liskov Substitution Principle

Posted on So 14 November 2021 in Patterns

Table of contents:
  • Intro

Intro

Liskov Substitution Principle states that if an interface takes some base class, it should work with every subclass without breaking the application. Therefore subclasses need to behave in "the same way" as their parent class. This is similar to the design by contract philosophy.

Benefits:

  • No undeterministic behaviour

Key:

  • Subclasses need to have less strict (or equally strict) rules as base class for this to work.

Examples:

  • Any Parent / Child relationship
  • Rectangle and Square example often given for breaking LSP

Short Forms:

  • LSP: Liskov Substitution Principle
