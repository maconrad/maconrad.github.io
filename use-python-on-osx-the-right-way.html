Installing and using pyenv and virtualenvwrapper

Installation python3 on OSX direcrtly and/or trying to make default python point to python3 on OSX is not a good idea. Worst case, it can break your OSX python installation.

In addition, what happens when you have to update from let's say python 3.7.3 to 3.9.7? Pyenv in combination with virtualenwrapper solves this issue by providing a python version manager (pyenv) as well as a python virtualenv manager (venv module could be used.)

In addition if working amonsgst a team pipenv which fixes dependencies in a lock file could be helpful. See references for more details.

brew install pyenv 
pyenv install 3.9.7
pyenv install 3.10.0
pyenv global 3.10.0
pyenv versions
$(pyenv which python3) -m pip install virtualenvwrapper
$(pyenv which python3) -m pip install --upgrade pip
source ~/.zshrc

Then adding the following to ~/.zprofile

# pyenv specific config
eval "$(pyenv init --path)"

and the following to ~/.zshrc

# Virtualenvwrapper specific config
export WORKON_HOME=~/.virtualenvs
mkdir -p $WORKON_HOME
. ~/.pyenv/versions/3.10.0/bin/virtualenvwrapper.sh

Afterwards we can use virtualenvwrapper like so:

mkvirtualenv <name>
workon <name>
# If project dir name and venv name are equal
workon . 
deactivate
rmvirtualenv <name>
lsvirtualenv

That's it. For more details see links below.
