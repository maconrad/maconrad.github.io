Intro

Open Closed principle states that new functionality should be added via extension, ot via modification. So what does that mean?

Benefits: - Scales well - Does not change interface how class is used - Prevents state/space explosion and code replication

Key: - New functionality is added via extension, not via modification

Example: - Lots and lots of filters, adding new filters = pain without OCP

Short Forms: - OCP: Open Closed Principle

Before

  • CarSearch Class is violating open closed prinicple
  • Note: No error handling is on purpose to keep code as lean as possible and still understand pattern. 
from enum import Enum

class Color(Enum):
    RED = 1
    GREEN = 2
    BLUE = 3

class Make(Enum):
    VOLVO = 1
    FIAT = 2
    AUDI = 3

class Car:
    def __init__(self, name, color, make) -> None:
        self.name = name
        self.color = color
        self.make = make

class CarSearch:
    def filter_by_color(self, cars, color):
        for c in cars:
            if c.color == color:
                yield c

    def filter_by_make(self, cars, make):
        for c in cars:
            if c.make == make:
                yield c

    def filter_by_color_and_make(self, cars, color, make):
        for c in cars:
            if c.color == color and c.make == make:
                yield c

if __name__ == '__main__':
    car1 = Car('DailyDriver', Color.RED, Make.FIAT)
    car2 = Car('WinterCar', Color.BLUE, Make.VOLVO)
    car3 = Car('NiceWeather', Color.BLUE, Make.AUDI)

    cars = [car1, car2, car3]

    searcher = CarSearch()
    print(f"Blue cars:")
    for car in searcher.filter_by_color(cars, Color.BLUE):
        print(f"{car.name}")

The problem is that each new filter adds new combinations resulting in a explosion of combinations. For example: - 2 Filters: byColor, byMake, byColorMake = 3 Combinations - 3 Filters, byColor, byMake, ByEngine, byCM, byCE, byME, byCME = 7 Combos

After

  • Uses specification Enterprise pattern
  • Very flexible with all() and map(func, iterator) functions
  • No need for any more methods, can only add new specifications if needed. E.g. EngineSpecification, TiresSpecification, DoorsSpec etc. etc.
  • OR can be implemented similarly using any() method
  • If there is need for e.g. a HouseSearch(), same specifications can be reused, e.g. ColorSpecification. Just create a new Search Class be inheriting from Search.
  • Note: No error handling is on purpose to keep code as lean as possible and still understand pattern.

```python from enum import Enum

class Color(Enum): RED = 1 GREEN = 2 BLUE = 3

class Make(Enum): VOLVO = 1 FIAT = 2 AUDI = 3

class Car: def init(self, name, color, make) -> None: self.name = name self.color = color self.make = make

Base classes

class Specification: def is_satisfied(self, item): pass

class Search: def filter(self, items, spec): pass

class ColorSpecification(Specification): def init(self, color) -> None: self.color = color

def is_satisfied(self, item):
    return item.color == self.color

class MakeSpecification(Specification): def init(self, make) -> None: self.make = make

def is_satisfied(self, item):
    return item.make == self.make

Combinator

class AndSpecification(Specification): def init(self, *args) -> None: self.specs = args

def is_satisfied(self, item):
    return all(map(
        lambda spec: spec.is_satisfied(item), self.specs
    ))

Could be in base Class, but more extendable like this

class CarSearch(Search): def filter(self, cars, spec): for car in cars: if spec.is_satisfied(car): yield car

if name == 'main': car1 = Car('DailyDriver500', Color.RED, Make.FIAT) car2 = Car('WinterCar', Color.BLUE, Make.VOLVO) car3 = Car('NiceWeather', Color.BLUE, Make.AUDI) car4 = Car('DucatoCamper', Color.RED, Make.FIAT)

cars = [car1, car2, car3, car4]

searcher = CarSearch()
print(f"Blue cars:")
for car in searcher.filter(cars, ColorSpecification(Color.BLUE)):
    print(f"{car.name}")

print(f"Volvos cars:")
for car in searcher.filter(cars, MakeSpecification(Make.VOLVO)):
    print(f"{car.name}")

print(f"RED Fiat cars:")
for car in searcher.filter(cars, AndSpecification(
    ColorSpecification(Color.RED),
    MakeSpecification(Make.FIAT)
    # Could be 20 specs...
)):
    print(f"{car.name}")
````
