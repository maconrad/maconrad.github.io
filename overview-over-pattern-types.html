Intro

GoF (Gang of Four) defined the following design patterns back in 1994 and categoriezed them into the following groups:

  • Creational
  • Structural
  • Behavioural

Solid Design Principle

Defined by Robert C. Martin and very often used as a reference. Main points here are:

  • Separation of Concern / Single responsibility
  • Open-Closed Principle
  • Liskov Substitute Principle
  • Interface segration Principle
  • Dependency Inversion Principle

Creational Patterns

The main patterns here are:

  • Builder
  • Factories (Abstract Factory & Factory Method)
  • Prototypes
  • Singleton

There are different types of creational Patterns:

  • Explicit via a constructor vs. implicit e.g. using DI or reflection
  • Step-by-step creation vs single statement creation

Structural Patterns

The main patterns here are:

  • Adapter
  • Bridge
  • Composite
  • Decorator
  • Facade
  • Flyweight
  • Proxy

Mainly deals with the structure of the class, i.e. its class members. Very often these patterns are wrappers that try to mimic the wrapped/underlying class.

Behavioral Patterns

The main patterns here are

  • Chain of Responsibility
  • Command
  • Interpreter
  • Mediator
  • Memento
  • Observer
  • State
  • Strategy
  • Template Method
  • Visitor

Each of them are quite unique and address different problems.
